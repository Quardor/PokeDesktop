# Исследование API для проекта PokeDesktop

## Выбранное API

**PokéAPI**  
**Ссылка:** [https://pokeapi.co](https://pokeapi.co)

### Почему выбран именно этот API:
- Не требует авторизации (API Key)
- Возвращает данные в формате JSON
- Большое количество полезных данных и изображений
- Стабильный и хорошо документированный
- Позволяет использовать минимум 2 эндпоинта
- Интересная тема (Покемоны)

## Изученные альтернативы

1. **JokeAPI** (`https://v2.jokeapi.dev`) — простой, но слишком примитивный
2. **Rick and Morty API** (`https://rickandmortyapi.com`) — хороший, но меньше данных о персонажах
3. **PokéAPI** — выбран как наиболее подходящий

## Используемые эндпоинты

### 1. Получение списка покемонов
- **Метод:** `GET`
- **URL:** `https://pokeapi.co/api/v2/pokemon?limit=20&offset=0`
- **Параметры:** `limit`, `offset`
- **Возвращает:** Список покемонов с именем и URL для детальной информации

### 2. Получение информации об одном покемоне
- **Метод:** `GET`
- **URL:** `https://pokeapi.co/api/v2/pokemon/{name}` (например, `pikachu`)
- **Возвращает:** 
  - `id`
  - `name`
  - `height`, `weight`
  - `types`
  - `abilities`
  - `stats`
  - `sprites`
